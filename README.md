# Java双端客户信息管理系统
## 项目简介
本项目是开源课程实践作业，基于Java面向对象分层开发，提供**控制台终端**、**浏览器网页可视化**两套操作入口，面向Java初学者、小型线下商户。
解决纸质客户档案杂乱、查询修改繁琐的问题，全程使用数组内存存储客户数据，无第三方框架依赖，纯原生JDK实现，适合学习分层开发、简易HTTP服务、前后端交互基础逻辑。

## 技术栈
- 运行环境：JDK 8
- 后端：原生Java面向对象编程、数组内存存储、JDK内置HttpServer网络服务
- 前端：内嵌HTML + CSS + JavaScript，无需额外前端文件、Tomcat服务器

## 项目文件结构
- Customer.java        // 客户实体类，封装姓名、性别、年龄等属性
- CustomerList.java    // 数据管理层，数组实现增删改查核心逻辑
- CustomerView.java    // 控制台菜单交互视图
- Utility.java         // 输入工具类，统一校验输入长度、数字、菜单选择
- HttpServerHandler.java // 网页 HTTP 服务，内嵌前端页面，提供 web 操作接口
- README.md            // 项目说明文档
- LICENSE              // MIT 开源协议

## 核心功能
1. 客户实体封装
私有化存储姓名、性别、年龄、电话、邮箱，提供构造方法与get/set；年龄自带0-150范围校验，统一方法输出客户完整信息。
2. 底层数组数据管理
封装全套CRUD逻辑：新增时判断数组容量，查询会拷贝数据保护原数组，按编号修改、删除客户，自动处理数组移位与下标越界校验。
3. 控制台菜单交互
循环主菜单支持添加、修改、删除、查看客户、退出五大功能；工具类统一校验输入格式与长度，删除、退出增加二次确认，降低误操作。
4. 网页可视化管理页面
内置JDK HTTP服务监听8080端口，内嵌带样式的前端页面；通过JS异步调用后端接口，网页表单完成增删改查，自动生成客户表格展示数据。
5. 通用输入校验工具
统一处理控制台输入，限制字符串长度、校验纯数字、菜单选项、确认选择，减少重复代码，交互提示统一规范。

## 完整运行教程
### 准备
1. 将项目内所有Java文件放在同一包，不能缺失任意类；
2. 使用JDK8运行，高版本JDK会出现HttpServer权限报错；
3. IDE无代码报错后再启动程序。

### 方式一：控制台终端版本（CustomerView）
1. 运行CustomerView的main方法，控制台弹出功能菜单；
2. 输入数字1添加客户，依次填写姓名、性别、年龄、电话、邮箱，非法输入会要求重新填写；
3. 输入数字2修改客户，输入客户编号，系统校验编号有效后录入新信息完成覆盖；
4. 输入数字3删除客户，填写编号并确认Y/N，确认后清除对应客户数据；
5. 输入数字4查看全部客户，表格展示所有客户完整信息；
6. 输入数字5选择是否退出，Y关闭程序，N返回菜单继续操作；
7. 关闭程序后所有客户数据清空。

### 方式二：网页可视化Web版本（HttpServerHandler）
1. 运行HttpServerHandler的main方法，本地8080端口服务启动成功；
2. 浏览器访问 http://localhost:8080 打开可视化管理页面；
3. 点击顶部按钮唤起对应表单：添加客户填写信息提交、查看客户自动生成表格、输入编号删除/修改客户；
4. 前端通过JS发送请求调用后端接口，页面下方实时展示操作成功或失败提示；
5. 网页程序与控制台程序不能同时运行，两者数据相互独立，重启服务数据清空。
